# Harden Surface - Hyperliquid Safety + Autonomy

## Ranked Targets
1. Hyperliquid validator replay: signatures must bind `intentHash`, `executionHash`, vault, action kind, score, deadline, and verifier domain.
2. HyperEVM deployment reality: stack pieces must deploy under per-transaction gas limits and CoreWriter encoding must match live fork state.
3. Live Hyperliquid readiness: live submission must fail before venue/API submission if kill switch is on or API wallet approval is not submitted.
4. Autonomous promotion: generated/candidate strategy changes must not promote from backtest-only evidence.
5. Overfit risk: in-sample wins must be rejected if out-of-sample walk-forward/paper evidence is weak.
6. Test-integrity gap: external Hyperliquid order submission cannot be run unconditionally in CI without funded testnet credentials and side effects.

## Invariants Added
- Hyperliquid direct validator signatures do not replay across execution hash, vault, or action kind.
- Hyperliquid stack deploy plus CoreWriter path works on a HyperEVM testnet fork with real system contract state.
- Candidate strategy promotion requires walk-forward approval and paper-trade evidence.
