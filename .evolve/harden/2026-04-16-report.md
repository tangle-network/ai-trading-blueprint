# Harden Report — Backtest Engine
Date: 2026-04-16

## Findings fixed (all in this commit)

| # | Severity | Finding | Fix |
|---|----------|---------|-----|
| 1 | CRITICAL | Per-token candles not sorted → silently dropped candles | Added `tc.sort_by_key(\|c\| c.timestamp)` after partition |
| 2 | CRITICAL | `Decimal::try_from(f64)` panics on NaN/Inf in trail_pct, activation_pct, level_price | Replaced all with `f64_to_decimal()` which guards NaN/Inf/huge values |
| 3 | HIGH | No rule count limits → CPU DoS via 100k entry_rules | `validate()` caps: 100 entry_rules, 50 exit_rules, 20 filters |
| 4 | HIGH | f64 fields (weight, fraction, kelly_multiplier) not range-checked | `validate()` checks NaN, Inf, bounds on all f64 fields |
| 5 | HIGH | VolatilityGate min_atr_pct > max_atr_pct not caught | `validate()` rejects inverted range |
| 6 | MEDIUM | Kelly fraction NaN when avg_loss near epsilon | Tightened guard to 1e-10, added b/result NaN/Inf checks |
| 7 | MEDIUM | `f64_to_decimal` panics on very large floats | Added abs() > 1e20 guard before Decimal conversion |
